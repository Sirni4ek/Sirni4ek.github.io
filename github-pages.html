Getting Started with GitHub Pages

To get started with GitHub Pages, follow these steps:

  1. Create a GitHub repository.
  2. Enable GitHub Pages in the repository settings.
  3. Select a publishing source, such as the main branch.
  4. Add HTML, CSS, and JavaScript files to your repository.
  5. Your website will be published at `https://your-username.github.io/repository-name`.

How GitHub Pages Works

GitHub Pages works by serving static files from your GitHub repository. When you push HTML, CSS, or JavaScript files to the specified branch, GitHub automatically deploys them to a web server.

Common Pitfalls

  • Forgetting to enable GitHub Pages in repository settings.
  • Not using the correct branch or folder (check the publishing source).
  • Missing or incorrectly named `index.html` file (this is the default homepage).
  • Not waiting long enough for changes to take effect (can take a few minutes).
