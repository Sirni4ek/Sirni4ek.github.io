Black-box testing xxx

Testing of something we cannot look inside of, including the source code.

White-box testing xxx When we can look inside the product, including the source code and database contents. Regression testing xx When we look for old problems to see if they have appeared again. Functional testing x Testing for the actual functionality the program is written to perform. Exploratory testing xx Random testing of whatever comes to mind, without a script. Usability testing xx Testing by a user, observed by developer. (We cannot do it by ourselves.) A/B testing xx Comparing how users react to different versions of the software, giving some users one version, and other users a different versions. Manual testing xxx Testing of the program by hand, just like a user would. Automated testing xx Writing of a program to test the program, either to test the code itself, or to simulate a user's actions. Integration testing x Checking the interactions between different components or systems. Smoke testing xx Quick cursory check of the system to ensure "no smoke", or nothing obviously wrong.
