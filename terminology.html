Terminology

Welcome

Thank you for visiting my website!

Black-box testing

Testing of something we cannot look inside of, including the source code.

White-box testing

When we can look inside the product, including the source code and database contents.

Regression testing

When we look for old problems to see if they have appeared again.

Functional testing

Testing for the actual functionality the program is written to perform.

Exploratory testing

Random testing of whatever comes to mind, without a script.

Usability testing

Testing by a user, observed by developer. (We cannot do it by ourselves.)

A/B testing

Comparing how users react to different versions of the software, giving some users one version, and other users a different version.

Manual testing

Testing of the program by hand, just like a user would.

Automated testing

Writing of a program to test the program, either to test the code itself, or to simulate a user's actions.

Integration testing

Checking the interactions between different components or systems.

Smoke testing

Quick cursory check of the system to ensure "no smoke", or nothing obviously wrong.
